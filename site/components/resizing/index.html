--- title: resizing layout: component ---

A critical feature of GUI based operating systems is the ability to resize a window.

Typical Appearance

Resizing is typically indicated by a change in the size of the affected window. When resizing with the mouse, the mousce icon will usually change from a pointer to multiple arrows.

Typical Behavior

Windows are typically resized by clicking one of the edges and dragging the mouse to the desired shape. They may alternatively be resized by accessing the window menu.

Events

The most relevant resize event is the resize event. A resize event is triggered when the size of the window has been changed.

Variants

Window resizing has remained relatively the same since the introduction of windows in GUI baed operating systems.

Priority Metrics

Since it is a critical aspect of any GUI based operating system, most of the priority metrics are vital in this component's design analysis. Users who intend to resize a window must have no confusion as to how to accomplish the task. Resizing follows the natural way of thinking by allowing the user to increase the size from the corners and sides of the window, making the process simple to learn (learnability). Also, since resizing follows the natural mental model, the way in which the user resizes windows is easily memorable (memorability).

Since window resizing corresponds to either mouse input or window menu input, the response is immediate (efficiency). Since window resizing is easily learnable and highly memorable, it is unlikely that users commit errors when attempting the task (errors).

Due to its simplicity and utilitarian nature, resizing does very little in the way of providing satisfaction to the user (satisfaction). However, this does not detract from the overall good design of window resizing.

Key Characteristics

Key characteristics of resizing include the changing of the mouse icon from a pointer to arrows along with the visible change in window size.

Feedback

For resizing, the window visibly grows larger or shrinks according to user input. In older operating systems, a rectangle is shown to represent the size the window will change to when the user stops dragging the mouse.

Platform Specific Instances

When resizing windows with a mouse in older operating systems, a rectangle would be drawn to indicate the resulting shape of the resize once finished dragging. For modern operating systems, the window dynamically changes size and content within conforms to the shape as the window is resized.
