Search components are constantly present in computational systems. Generally providing an interface for text-based inputs, their main purpose is to help users finding files, programs and other resources in an easier way.

Typical Appearance

Search components have their style changed from one system to another, but, in a general way, its main idea remains the same. As we can see in the images below, the search bars are basically composed by a space where users can enter a text-based input and a button they can click to initiate the search:


Previous Next

Purpose

Search components allow users to quickly find a value in a large collection and also (depending on the search engine) create a filter to facilitate the search and make it more precise.

Typical Behavior

Basically, search components receive an input (the text-based input mentioned before), the user presses the search button (generally identified by the picture of a magnifying glass or by the word "search") and then, the results of the research are shown.

Events

As search boxes are composed by a text field and a button, the most relevant search events are enterText, clear and click. Secondary events include dragAndDropText.

Key Characteristics

What makes a search a "good" search? Perhaps consistency is the most important interaction design principle that a search must follow. Users must know how to put their input in a search the moment they see it. For this reason, a search should have an identical icon and layout.

Variants

Search has variations in its behavior. Despite their very similar appearances, their slight behavioral distictions create a big difference in efficiency.

Search on a web browser

There are two different types of searches on web browsers: an address bar and a search box. They behave in the same manner as a typical search box, but an address bar helps improve efficiency if users know the exact URL of the site they are looking for. An address bar can directly go to a site when users input the site URL.


A find bar

A find bar, the search box that looks for text within a document, has interesting user behavioral and interface differences from a search box for web browsers. Users utilize this tool to jump to where the word is located, occurs on the page as soon as they start typing. It has up and down arrow bottons instead of a search botton. The arrow buttons guide users to search for other place the word occurs. A find bar saves users' time in finding what they want in the page.

A system-wide search box and a system-wide search box

These search boxes look for all the files on a computer. They are simply two different ways of initiating searches. Although, some advanced system-wide search boxes such as the spotlight in OS X Yosemite searches the web as well.


Priority Metrics

The usability metrics are essential for search because every single user utilize this component. Users have to recognize what to do with a search the moment they see it on any platform and should not have any problems (learnability).

All the search components should help improve user performance (efficiency). For instance, users invoke a find bar to save time. Therefore, if a search requires multiple clicks or is not instantly recognizable, then its design cannot be considered efficient.

Example of the Search Bar

According to the Apple's Guidelines, a search bar can display optional elements, such as these:

Placeholder text: The text showed in the element should state the function of the control (for example, 'Search' as shown above) or remind users in what context they are searching (for example, 'Google') so they can execute their search in a more accurate way.

The Bookmarks button: This button can provide a shortcut to information users want to easily find again. For example, the Bookmarks button in the Maps search mode gives access to bookmarked locations, recent searches, and contacts.

Example of the Bookmark button

The Clear button: Most search bars include a Clear button which allows users to clear the contents in the text space in a single step. The button shows up when the user enters an input and hides when there is no text in the text field.

Example of the Clear button

The results list icon: This icon indicates the presence of results from a search done by the user. When users tap the results list icon, an application can display the results of their most recent search.

Example of the results list icon

A prompt: A descriptive title (prompt) can be placed above the search bar. A prompt is a short, complete sentence that provides an introduction or an specific information for the search bar.

Example of the prompt feature

Apple's Spotlight

Spotlight is a resource provided by Apple that helps users quickly find anything on MacBook, including documents, emails, apps, songs, contacts, and more. Spotlight also provides suggestions (Spotlight Suggestions) from various sources (Wikipedia, Bing, Maps, news, and iTunes) so the user can have a complete set of information without leaving the app. Search results are composed by rich and interactive previews so the user can play song previews, get directions, send email, make phone calls, and more, directly from the results.

Priority Metrics

As said before, the usability metrics are essential for search because there are users that deal with this feature. Learnability, as shown, is an important metric when talking about search components; users should recognize the elements that compose the search box and know what to do.

Also, all the search components should help improve users' performances (efficiency) and turn the search experience as profitable as possible. That is what Apple's search functions try to do; provide the user with a complete environment that helps to find things faster and in a more accurate way.

Key Characteristics - Strive for consistency

Maybe consistency is the most important interaction design principle that a search must follow. Users must know what to do when facing a search component and, besides, the results shown by the search process must be accurate and consistant too.

    As previously mentioned, the usibility metrics for the search components do not change. The main ones are effiecncy and learnabilty. Key Components

What makes a search a "good" search?

Consistency and Efficiency

    Consistency is obviously number one in making a search component "good". And if we use Apple products as an example, they do well in keeping the search component consistent. Efficiency is the next key component, because that is the point of the search; to find information fast. However, in the example of Siri, consistency was devalued below efficiency, even though it is usually consistency which is held to the highest standard with search components. So when asking what makes a search "good"; both consitency and efficieny must be in the answer.
Ubuntu

Ubuntu

When looking at platform-specific instances of Search on Ubuntu, we find that all instances lie within category of application and file searching. We will be looking at three of Ubuntu's search capabilties (Search your computer, Files search, and Nautilus desktop search) and how they are different from their Windows and OS X alternatives.

Search your computer

We have already looked at OS X's Spotlight Search and Window's Equivalent Taskbar Search and all of their features, and although Ubuntu's Search your computer is missing some features that both Window's and OS X offer, it is generally the same tool. You can use it to search within your own computer for anything and access it in the fastest way possible. You use the Windows Button to easily access this tool and then instantly start to type the name of anything you are looking for and click enter to load it up.

One key differences between the three version of the tool is that they all different placements of the tool. For example, Windows has decided that the best place for the tool is the bottom left, and now every Windows' user has become comfortable with that placement. OS X on the other hand decided that it should be a popover and that it should be in the middle of the screen. Ubuntu is also unique it their approach. They are the only one that decided that the tool should be fullscreen. When you open Search your computer it opens up a window that covers the entire screen.

Previous Next

Search your Computer Tool

As you can see in the slideshow above, there are five different types of search within the Search your computer tool: General Search, Application Search, Files and Folders Search, Music Search, and Photo Search.

The General Search encompasses all of the other options and shows them together on one page. For example, if one were to type the query "Ubuntu", the general tool is able to search the entire computer and display the results as follows:

Using the General Search Tool to search for Ubuntu

As you can see, with the query "Ubuntu" the General Search Tool has split the search results into Applications and Files & Folders, and we are able to see all of these results on one page. You can then select whichever application/file that you were intending to find, and you will be sent there.

In this example, I am using a fresh Virtual Machine and therefore don't have much of a problem finding anything when there are only ten items to choose from, but let's pretend that we were searching for a file from your class "Biological Databases" and you don't exactly remember what you named it. If one hundred files were displayed after the search, you could use the filter tool to slim down the search results, and make it a bit easier to find the file you are looking for.

Closeup of Filter Options within General Search Tool

As you can see, their filter tool comes with options beyond the five categories that are listed above. For example, you could restrict the search to look for only files that are code related. This is a very handy tool for developers, because you can search a folder and then click the code filter and it will show all the code within that folder. There is also an option to filter by source. The filter bar lists possible sources of the file. This means you can restrict the search space to files that you worked on in a specific application.

This tool is the fastest way to search and access anything within your computer. You would be constantly using it to run applications, find and open files, and search for media. Unlike it's Windows and OS X counterparts, Ubuntu's Search you computer tool is unable to search the internet or perform calculations. Regardless, it is a very intuitive and helpful tool to use.

Files Search

Although you could, and would, use the Search your computer tool to find files in the fastest manner, Ubuntu still offers the traditional method of locating files through the File Manager. Once you have the File Manager open, you can use the Search icon to lower a search bar which allows you to search for files within the current directory. Their File Manager is the same as Windows and OS X with very subtle differences, most of which pertain to stylistic choices. All three operating systems' File Manager do the same thing.

File Manager Search Activated

With the search bar open, you are able to type anything to search for it within your directory, but once again, if you have hundreds of files to search through and there are a lot that match the query, Ubuntu offers a filter here also. You are able to filter your query to search for files specifically (i.e. filter out any executables or applications) and then filter even more by selecting a period of time. This period of time, for example 1 day ago, would filter the search to files that were created within the past day.

File Manager Search with Filter
